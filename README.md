# Аналіз утримання користувачів за допомогою когортного аналізу

## Опис проєкту

У цьому проєкті проведено аналіз утримання користувачів — **Retention Rate** — за допомогою когортного аналізу.

Основна мета проєкту — визначити, як змінюється активність користувачів протягом перших місяців після реєстрації, а також порівняти утримання користувачів, залучених через промоакції, з користувачами, які прийшли органічно.

Для підготовки та очищення даних використовувався **SQL**, а для створення когортних таблиць, розрахунку Retention Rate та візуалізації результатів — **Google Sheets**.

## Дані

У проєкті використовувалися дві таблиці.

### `cohort_users_raw`

Містить інформацію про користувачів:

- унікальний ідентифікатор користувача;
- ім’я та прізвище;
- електронну пошту;
- країну;
- дату та час реєстрації;
- канал залучення;
- пристрій;
- ознаку промозалучення.

### `cohort_events_raw`

Містить інформацію про активність користувачів:

- унікальний ідентифікатор події;
- ідентифікатор користувача;
- дату та час події;
- тип події;
- суму транзакції.

Дати реєстрацій і подій були записані в різних текстових форматах, тому перед аналізом їх було очищено та приведено до єдиного формату.

## Етапи виконання проєкту

### 1. Очищення даних у SQL

За допомогою SQL було виконано:

- видалення зайвих пробілів;
- заміну різних розділювачів у датах;
- приведення дат до єдиного формату;
- обробку двозначного та чотиризначного формату року;
- видалення записів із відсутніми датами;
- видалення подій без зазначеного типу;
- виключення тестових подій.

### 2. Об’єднання таблиць

Дані про користувачів були об’єднані з даними про їхню активність за полем `user_id`.

У результаті для кожного користувача було визначено:

- місяць реєстрації;
- місяць активності;
- кількість місяців після реєстрації;
- належність до промо- або органічної групи.

### 3. Розрахунок когорт

Для аналізу були створені такі показники:

- `promo_signup_flag` — ознака промозалучення користувача;
- `cohort_month` — місяць реєстрації користувача;
- `activity_month` — місяць активності користувача;
- `month_offset` — кількість місяців між реєстрацією та активністю;
- `users_total` — кількість унікальних активних користувачів.

Період аналізу охоплює активність користувачів із січня до червня 2025 року.

### 4. Побудова когортних таблиць

Підготовлені в SQL дані були експортовані до Google Sheets.

У Google Sheets створено:

- таблицю з кількістю активних користувачів за когортами;
- таблицю з показником Retention Rate;
- умовне форматування для візуального порівняння когорт;
- зріз для перемикання між промо-, органічними та всіма користувачами;
- окремий аркуш із висновками.

## Розрахунок Retention Rate

Показник утримання користувачів розраховувався за формулою:

```text
Retention Rate =
Кількість активних користувачів у місяці N
/
Кількість користувачів у місяці 0
× 100%
```

Для нульового місяця кожної когорти показник Retention Rate дорівнює 100%.

## Структура Google Sheets

Файл містить три основні аркуші:

- `Data` — підготовлені дані з SQL;
- `Cohort_tables` — когортні таблиці та Retention Rate;
- `Conclusions` — висновки за результатами аналізу.

## Основні результати

Когортний аналіз дає можливість:

- оцінити, як довго користувачі залишаються активними після реєстрації;
- порівняти якість промо- та органічного залучення;
- визначити когорти з найвищим і найнижчим рівнем утримання;
- оцінити ефективність каналів залучення;
- виявити зміни в поведінці користувачів протягом перших місяців.

Порівняння промо- та органічних користувачів дозволяє оцінити не лише кількість залучених клієнтів, але й якість цього залучення.
